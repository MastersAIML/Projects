Right: % is a multi-character wildcard whereas _ is single character wildcard. For example, “_r%” will mean words with second character as r followed by zero or more characters at the end.

The having clause is typically used when you have to apply a filter condition on an 'aggregated value'. This is because the 'where' clause is applied before the aggregation takes place, and thus it is not useful when you want to apply a filter on an aggregated value.

 

In other words, the having clause is equivalent to a where clause after the group by has been executed but before the select part is executed. You can read this stackoverflow answer to understand the having clause better.
